# Restaurant Delivery SMO System
Проект моделирует систему обработки онлайн-заказов в ресторане доставки еды как  
систему массового обслуживания (СМО). Модель учитывает реалистичные ограничения:  
ограниченную очередь на приготовление (буфер), случайное время готовки блюд и  
высокую нагрузку в пиковые часы.

### Ключевые особенности:

-> Заказы поступают равномерным потоком (И32) от бесконечного источника клиентов (ИБ). <br>
-> Обслуживание выполняется тремя кухонными линиями с экспоненциальным временем приготовления (П31).  <br>
-> При занятости всех линий заказы помещаются в буфер по кольцу (Д1ОЗ1). <br>
-> Если буфер переполнен, система автоматически отменяет самый старый заказ (Д1ОО3), так как он теряет актуальность.<br>
   

## Model Parameters
**-ИБ:** Источник (бесконечный) <br>
**-И32:** Источник 3, закон распределения (равномерный-ИЗ2) <br>
**-П31:** Прибор 3, закон обслуживания (экспоненциальный-ПЗ1)<br>
**-Д1031:** Дисциплина постановки в буфер (по кольцу-Д1ОЗ1)<br>
**-Д10O3:** Дисциплина постановки в буфер (по кольцу-Д1ОЗ1)<br>
**-Д2П2:** Дисциплина выбора прибора заявка идёт на первый свободный прибор<br>
**-Д2Б2:** Дисциплина выбора заявки из буфера (в порядке поступления “FIFO”)<br>
**-OР2:** Отображение результатов (графики по значениям параметров)<br>
**-ОД2:** Динамическое отображение (формализованная схема модели и текущее состояние)<br>
